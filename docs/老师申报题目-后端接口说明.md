# 老师申报题目 - 后端接口说明

前端「老师申报题目」页面依赖以下数据与接口，请后端按此实现或核对。

---

## 1. 数据表与视图

### 1.1 导师已选实习项目列表

**需求**：校内导师在「老师申报题目」页需要看到**自己已选的实习项目**，数据来源为：

- **表 31 RelIntershipUser**：用户与实习项目关联（UserId, InternshipId）
- **表 16 MainInternship**：实习项目主表

**后端需提供视图（或等价接口）**：

- **视图名**：`ViewTeacherSelectedInternship`（或与前端约定一致的 keyWords）
- **含义**：当前用户（导师）已选的实习项目，即 RelIntershipUser 与 MainInternship 的关联结果。
- **建议 SQL 思路**：
  ```sql
  SELECT r.InternshipId, r.UserId, m.Id, m.Name AS internshipName,
         m.StartTime AS startTime, m.EndTime AS endTime
  FROM RelIntershipUser r
  INNER JOIN MainInternship m ON m.Id = r.InternshipId
  WHERE r.UserId = :userId
  ```
- **查询条件**：前端会传 `searchKey: { userId: 当前登录用户ID }`。
- **返回字段要求**：至少包含 `id`、`internshipId`（或与 MainInternship 主键一致）、`internshipName`（或 `name`）、`startTime`、`endTime`，供项目选择与标题展示使用。

若使用 `getSomeRecords` 的 keyWords 机制，需在后台配置该视图/关键字与上述查询逻辑。

---

### 1.2 表 15 RelTeacherStudent（导师学生选择表）

**用途**：每条记录对应「一个题目」，老师申报题目时写入/更新该表。

**前端使用方式**：

- **列表**：使用视图 `ViewRelTeacherStudent`，查询条件为 `teacherId = 当前用户ID`、`internshipId = 当前选中的实习项目ID`。
- **新增/编辑**：通过 `dataList/editOneNode`，keyWords 为 `RelTeacherStudent`，提交字段见下。

**表结构需支持**（与现有表 15 的兼容）：

| 字段           | 说明           | 说明 |
|----------------|----------------|------|
| id             | 主键           | 编辑时必传 |
| teacherId      | 导师 ID        | 必填，关联表 1（用户表） |
| StuId          | 学生 ID        | 申报题目阶段可未分配，建议允许 0 或 NULL（若表结构为 NOT NULL 则用 0 表示「未分配」） |
| InternshipId   | 实习项目 ID    | 必填，关联表 16 MainInternship |
| **name**       | 题目名称       | **必填**，老师申报的题目名称，前端会传 `name` 字段 |

若当前表 15 没有 `name` 字段，需要在表中**新增字段 `name`**（如 `VARCHAR(200)`）用于存储题目名称。

**前端提交示例**：

- 新增题目：`{ teacherId, internshipId, name, stuId: 0 }`（若后端允许 StuId 为空则可省略 stuId）
- 编辑题目：`{ id, teacherId, internshipId, name }`

## 2. 接口清单

以下均基于现有通用列表/编辑接口，只需保证 keyWords 与视图支持上述查询与字段即可。

| 接口/用途           | keyWords                     | 说明 |
|--------------------|------------------------------|------|
| 导师已选项目列表   | `ViewTeacherSelectedInternship` | searchKey: `{ userId }`，返回该项目列表 |
| 题目列表查询       | `ViewRelTeacherStudent`      | searchKey: `{ teacherId, internshipId }`，需返回 `id, name, internshipName, studentName` 等 |
| 题目新增/编辑      | `RelTeacherStudent`          | editOneNode，传入 `id`（编辑时）、`teacherId`、`internshipId`、`name`，必要时 `stuId: 0` |
| 题目删除           | `RelTeacherStudent`          | delOneOrManyNodes，传入题目记录 id 数组 |

## 3. 视图 ViewRelTeacherStudent 字段建议

为支撑「老师申报题目」列表展示（题目名称、实习项目、**创建人**、创建时间、**状态**、操作），视图中**必须**包含以下字段：

- `id`：RelTeacherStudent 主键  
- `teacherId`、`internshipId`、`stuId`  
- **`name`**：题目名称（来自表 15 的 name 字段）  
- `internshipName`：实习项目名称（来自 MainInternship）  
- **`createUserName`**：创建人姓名（用于表格「创建人」列；可关联用户表根据 createUserId 取 name，若表无 createUserId 则用 teacherId 关联）  
- `createTime`：创建时间（表格「创建时间」列，若为公共字段则已有）  
- **`isAudit`**：审核状态（用于表格「状态」列；取值与系统约定一致：-1 待提交、0 待审核、1 审核通过、2 审核不通过、3 审核退回；若题目暂无流程则可为 -1 或 0）  
- `studentName`：学生姓名（可选，来自用户表，stuId 为空/0 时可显示为空或「未分配」）

---

## 4. 老师申报题目列表与提交（与实习计划/企业岗位申报一致）

前端「老师申报题目」列表使用 **ViewVerifyProcessRelTeacherStudent**（关联 MainVerifyProcess + RelTeacherStudent），与实习计划制定、企业岗位申报同一套逻辑：

- **列表**：`keyWord.view = 'ViewVerifyProcessRelTeacherStudent'`，查询条件 `searchKey = { internshipId, teacherId }`。每行：`id` = MainVerifyProcess 主键，`relationId` = RelTeacherStudent 主键（题目 id），`isAudit`、`name`、`internshipName`、`teacherName`、`createTime` 等。
- **提交**：点击「提交」时调用 `POST /internshipProcess/auditProcess`，请求体 `node` 包含：`id`（当前行 MainVerifyProcess 的 id）、`isAudit: 0`，以及可选 `processId`、`relationId`、`createUserId`、`tableName: 'RelTeacherStudent'`。
- **删除**：仅允许 `isAudit === -1`（待提交）时删除；前端传 `RelTeacherStudent` 的 id（即列表行的 `relationId`）给 `delOneOrManyNodes('RelTeacherStudent', ids)`，后端在删题目前按 `deleteVerifyProcessByRelationIdAndTableName(id, 'RelTeacherStudent')` 清理审核记录。
- **修改**：仅当 `isAudit === -1` 或 `3`（退回）时显示「修改」；编辑时用行的 `relationId` 作为题目 id 打开弹窗。

## 5. 小结

1. **ViewTeacherSelectedInternship**：按 `userId` 查询导师已选实习项目（RelIntershipUser + MainInternship）。  
2. **RelTeacherStudent 表**：增加 **name** 字段存题目名称；StuId 在申报题目阶段可为 0 或 NULL。  
3. **ViewVerifyProcessRelTeacherStudent**：列表用此视图，支持按 `teacherId`、`internshipId` 查询；返回 `id`（流程 id）、`relationId`（题目 id）、**isAudit**、name、internshipName、teacherName、createTime 等。  
4. 提交走 `auditProcess`，删除走 `delOneOrManyNodes('RelTeacherStudent', ids)`（后端负责清理 MainVerifyProcess）。
